What's New · Kingdom of Caid
Updated in sync with the demo · Last updated May 30, 2026 · 11:25 PM PT
Thanks for testing! This page gives you a plain-English rundown of what changed and what to click on. You don't need any technical background — just try the things listed under "Give it a try" and let Rayhana know if anything feels off or confusing.
⚜️ Explore the App Start here if you're new
As an attendee
Browse the catalog. Use the search bar or the dropdowns to filter by topic, format (in-person or online), or event. Do the results make sense?
Register for a class. Click any class card and hit Register. You'll be asked for your name and email — this is how the system tracks your spot.
Try joining a full class. Find one that says "Full" or has a waitlist. You should be offered a waitlist spot and told your position.
Check My Schedule. After registering for a couple of classes, click "My Schedule" in the top menu. Everything you signed up for should appear. Try the Print button.
As a teacher
Submit a single-day class. Click "Teach a Class" and select Collegium as the event. Fill in a title, description, prerequisites, and a max class size. Submit it — then log into admin (password: admin123) and confirm it shows up under Pending with everything you entered.
Add a teacher bio. While filling out the form, find the optional bio field and write a sentence or two about your background. After submitting, go to the catalog and find your class — hover over the teacher name. A small info bubble with your bio should appear.
Submit an online class. Select Online / Zoom as the format. A field will appear asking for your Zoom link — paste any URL in there. Submit and check that the link shows up correctly on the class card in the catalog.
Teach the same class more than once at Great Western War. Submit a class for Great Western War and look for the "How many times?" field — set it to Twice (or 3+). Use the day-by-day scheduler to pick your preferred days and times. In admin, the pending entries should appear labeled Session 1 of 2 and Session 2 of 2 so the coordinator knows they go together.
As an admin or coordinator
Log in. Click the ⚙️ gear icon in the top-right corner. Password is admin123. This is where the A&S officer, Seneschal, or event coordinator would work.
Create an event. Go to Admin → Events → Create Event. Enter a name, dates, and a number of classrooms — the system will generate Room 1, Room 2, etc. automatically. You can rename them later. Does the form make sense?
Publish and unpublish an event. Find the Published toggle on an event and turn it off. Go back to the catalog and confirm that event's classes disappear from public view. Turn it back on and confirm they return.
Approve a class. Go to Admin → Classes → Pending. Click Approve on any class and fill in the date, time, and room. Submit — then confirm the class moves to the published list and appears in the catalog.
Reject a class. From the Pending tab, reject a submission. Confirm it disappears from the queue.
Edit an approved class. Find a published class and click the edit (pencil) button. Change the room or time and save. Confirm the update shows correctly in the catalog.
Cancel a class. Click the cancel button on a published class. Confirm it shows as Cancelled in the catalog and on any registered attendee's My Schedule.
Check the schedule grid. After approving several classes, open the schedule grid view. Approve two classes into the same room at the same time — the grid should flag the conflict. For GWW, use the day tabs to switch between days.
View a class roster. Click the roster icon on any approved class with registrations. You should see who's signed up. Try printing it.
Community features
See who's going. Register for a class, then open that class card again. Your name should appear in the attendee list. This is how members can see who else signed up.
Register anonymously. When registering for a class, check the option to hide your name from the roster. Confirm your name appears as "Anonymous attendee" in the attendee list instead.
Upvote a wishlist request. Click the heart on a few wishlist items. The count should go up. Try voting on the same item twice with the same email — it should only count once.
Submit a class request. Add something to the wishlist. Try checking the option to hide your name, then find your request in the list and confirm it shows anonymously.
Drop a class. Register for a class, then remove yourself from it. Confirm your spot is freed up and your name is gone from the roster.
See a cancelled class on your schedule. Register for a class, then have an admin cancel it. Go to My Schedule — the class should still be listed but marked with a red Cancelled badge.
General notes
Data resets on refresh. This is expected — the demo doesn't have a real database behind it. Anything you do (registrations, submissions, admin changes) is temporary and clears when you refresh the page.
Class photo upload doesn't fully work yet. You can choose a photo when submitting a class, but the demo has no place to store it — so the image is never actually saved. It won't appear in the admin panel or on the class card. This is a known limitation being worked on.
Admin password is admin123. Click the ⚙️ gear icon in the top-right corner to log in.
May 30 Latest update 7 changes
Scheduling dates pre-fill correctly in the admin approval screen
Why it matters When an admin approves a class, the date was showing today instead of the event date — admins had to fix it by hand every time. It now starts on the right day.
Give it a try Log into admin (password: admin123), go to the Pending tab, click Approve on any class, and check that the date already shows the event's start date.
Time fields now only allow clean 15-minute steps
Why it matters Previously you could type in odd times like 10:07. Now every time picker in the app snaps to the quarter-hour — 9:00, 9:15, 9:30, 9:45 — so schedules stay consistent.
Give it a try In admin, open Create Event or the Approve screen and click on any time field — the options should step in 15-minute increments only.
Create Event form now shows hints inside each blank field
Why it matters The form used to show empty boxes with no clue what to type. Now each field has light grey hint text inside it showing the expected format or an example.
Give it a try Go to Admin → Events → Create Event. Before typing anything, read the grey text inside each box — does it make sense? Does anything still feel unclear?
Event location links now go to the real SCA Caid places page
Why it matters Placeholder links were going nowhere. Now they point to the actual SCA Caid site so you can see how a real event link would behave.
Give it a try Open any class in the catalog and look for the event or location link. Click it — it should open the SCA Caid website in a new tab.
Teachers can now add notes for the admin when submitting a class
Why it matters Before, there was no way for a teacher to flag special needs — like needing A/V equipment, a large room, or specific setup. Now there's a free-text notes box at the bottom of the submission form.
Give it a try Click "Teach a Class" in the top menu, fill out the form, and look for the notes field near the bottom. Type something in it and submit — then log into admin and check if the note shows up on the pending class.
Wishlist voting now requires an email address
Why it matters Previously you could heart a request without identifying yourself at all. Now the first time you vote in a session, you'll be prompted for your email. It's used for admin contact only and never shown publicly.
Give it a try Go to Class Requests and click the heart on any item. A prompt should appear asking for your email before the vote goes through.
Voter emails visible to admin per request
Why it matters Admins can now see who voted for a request and reach out when a matching class gets scheduled. In the real version this goes to Google Sheets; in the demo it shows in a popup.
Give it a try Vote on a request, then log into admin and go to the Requests tab. Find that request and click "See emails" — your email should appear in the list.

Notes Good to know
Admin password
Access Click the ⚙️ gear icon in the top-right corner of the demo. Password is admin123. This unlocks the Events, Classes, and Requests tabs.
Data resets when you refresh
Known limitation This is expected behavior in the demo. Everything — registrations, submitted classes, admin changes — is temporary. Refreshing the page wipes it back to the starting state. The real version would save to a shared database.
Data may clear unexpectedly
Known bug In some cases, data may disappear without a refresh — especially after a longer session or switching between tabs. If something you did seems to have vanished, that's likely why. Note what you were doing and let Rayhana know.

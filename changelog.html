Volunteer Module — Backend Wiring & Seed Separation
New volunteer.html Architecture
June 2, 2026 · 12:45 PM PT
seed.js — Shared Demo Data File

All demo seed data has been extracted from index.html and volunteer.html into a single shared seed.js file. Both pages load it via a <script src="seed.js"> tag. Events, classes, teachers, registrations, volunteer shifts, verifiers, and class requests all live here now.

Why it matters: the two modules share the same events and volunteers — having one source of truth prevents the data from drifting out of sync. When the live backend is ready, seed.js stays in the repo but goes inert because IS_DEMO is false.
Give it a try: Open either file. The demo data populates exactly as before — nothing visible changed. The difference is under the hood.
volunteer.html — Full Backend API Wiring

The volunteer module is now fully wired for a live Google Apps Script backend. Every action — browsing shifts, claiming, dropping, verifying, and all admin operations — calls api(), which hits the live URL when configured or falls back to the in-memory demo. The config is a single line: VOL_SCRIPT_URL = 'YOUR_APPS_SCRIPT_URL_HERE'.

Why it matters: this is the critical path before GWW deployment. Swap the URL string and the module goes live with no other code changes.
No-Login Claim Flow

The separate Sign In modal has been removed. Instead, the Claim Shift modal collects SCA name and email inline — both required. Email is the unique identifier, consistent with the class registry. After claiming, the app automatically switches to My Shifts with the email pre-filled so you can see your roster immediately.

Why it matters: no login system means no accounts to manage, no passwords to forget, and no friction for walk-up volunteers. Email still provides persistent identity — enter it in My Shifts from any device to see your shifts.
Give it a try: Select Great Western War, click Claim Shift on any open shift. Enter an SCA name and email — the app claims the shift and drops you straight into My Shifts. Then use margery@example.com in the My Shifts lookup to see pre-populated shifts with verified and unverified states.

  • My Shifts is now a nav button (was hidden); uses email lookup instead of session state
  • Dead nav links (Catalog, My Schedule, Requests, Teach) removed from volunteer.html topbar — those live in index.html
  • Admin volunteer list now shows email + verified shift count per person
  • Backend design document added to repo: scheda-volunteer-backend.md

Volunteer Module — Frontend Foundation
New volunteer.html
June 2, 2026 · 11:33 PM PT
volunteer.html — New Module

A fully designed volunteer scheduling module, separate from the class registry but sharing the same visual language. Volunteers can browse open shifts by event, filter by day, department, and activity level, and claim shifts with one tap. The Happening Now view surfaces shifts starting within the next 2 hours — designed for walk-up volunteers at the gate.

Includes a ticket/raffle system: verified shifts earn 1 ticket per hour (double for high-need shifts). Volunteers enter a 4-digit PIN from their shift supervisor after completing a shift to mark it verified.

Why it matters: GWW needs a volunteer coordination system for Veterans Weekend. This module runs alongside the class registry on the same repo, served from the same event records.
Give it a try: Open volunteer.html. Select Great Western War. Browse shifts, filter by department or activity level. Check the Now tab — it simulates Oct 9 at 10:00 AM to show Water Bearer and Chirurgeon Support as open. Admin PIN is 1234; verifier PINs are 4471, 8823, 3156.
Also in this update
  • Unified nav structure across both files — dead links styled as coming-soon
  • Shared CSS design tokens (colors, radius, shadow) between index.html and volunteer.html
  • Admin panel: Shifts, Volunteers, Verifiers, Dashboard tabs
  • PIN collision detection when adding verifiers
  • Double-ticket flag per shift for hard-to-fill roles
Room Schedule Printout & Roster Improvements
New Fix
June 1, 2026 · 12:10 AM PT
Room Schedule Printout

Admin > Events > Rooms now has a Print room schedules button that generates one page per room — the document you post on the door. Each page has a full-width dark blue header with the room name, followed by all approved classes in time order with title, teacher, duration, description, and tags.

Why it matters: this is the posting document for the day of the event. Teachers use the class roster (below); room monitors and attendees use this.
Give it a try: Admin > Events, expand Collegium 2026, click Print room schedules. Use your browser's Print or Save as PDF.
Class Roster — Teacher Check-In Tool

Printed rosters now include blank rows equal to the number of open enrollment slots (for walk-in hand-written additions), plus the class description and audience/level tags in the header.

Why it matters: teachers use this at the door to check in students and add walk-ins by hand. It's not a posting document — the room schedule printout handles that.
Give it a try: Admin > Class Grid, click Print Roster on any approved class. Barony/Location column was removed since that data was never collected.

Request Board, Admin Parity & Online Class Polish
New
May 30, 2026 · 11:25 PM PT
Class Request Board

Attendees can now submit class requests — topics they want taught — and upvote existing ones. The board is public; coordinators can hide or delete requests from the admin panel. Requests include submitter name, barony, and a "hide from public" flag.

Why it matters: helps coordinators recruit teachers for in-demand topics and gives the community a voice in programming.
Give it a try: Click Requests in the nav. Upvote a topic or submit a new one. Admin > Requests to manage the board.
Online Class Support

Classes with a meeting link now show a prominent Join Online button in My Schedule and the catalog. Online classes have a blue left border and tinted background to distinguish them at a glance. All "Zoom" references replaced with generic "Online" / "Meeting link" language.

Give it a try: Register for an online class, then check My Schedule for the Join Online button.
Also in this update
  • Admin edit modal now has full field parity with the teacher submission form (topic, format, duration, prerequisites, bio, image URL, meeting link)
  • My Schedule cards show prerequisites, fee, and duration
  • Request form: submitter SCA name required, barony optional, event pre-selects from URL param
